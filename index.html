The Dining Philosophers Problem

    Voraussetzungen



  • Es liegt eine Gabel zwischen den Teller

    Essen nur mit zwei Gabeln

    Jede Gabel wird einzeln aufgenommen

Probleme

deadlock

(Deadlock)

Jeder Philosoph wartet auf seine Rechte Gabel, diese ist aber durch seinen rechten Sitznachbarn belegt.

starvation

(Verhungern)

Jeder Philosoph wartet unendlich lange auf die rechte Gabel, da keiner seine linke Gabel ablegt.

concurrency

(Nebenläufigkeit)

Zwei Philosophen greifen gleichzeitig auf die selbe Gabel zu.

Pseudocode



    denken();
    linkeGabelAufnehmen();
    rechteGabelAufnehmen();
    essen();
    linkeGabelAblegen();
    rechteGabelAblegen();

Lösungen



Ressourcen Hierarchie

Der Kellner

Chandy/Misra Lösung

Resourcen Hierarchie

  • Dijkstra's urspüngliche Lösung
  • Jeder Philosoph nimmt zuerst die niedrigste zur Verfügung stehende Gabel auf


Vorteile:

  • Vermeidung von Deadlocks


Nachteile:

  • Ineffizient, vorallem bei unbestimtmer Anzahl von Ressources
  • Mögliche "Starvation"

Der Kellner

  • Ressources Verwaltung durch aussenstehende Instanz
  • Jeder Philosoph holt sich vom Kellner die Erlaubnis, Gabeln aufzunehmen
  • Der Kellner erteilt immer nur einem Philosophen gleichzeitig eine Berechtigung
  • Ebenfalls druch "Mutex"/Synchronisation umsetzbar


Vorteile:

  • Vermeidung von Deadlocks und "Starvation"


Nachteile:

  • Vermindert parallelen Ablauf der Threads

Chandy/Misra Lösung



  • Enteht aus einem Algorithmus zur Vermeidung von Deadlocks
  • Philosophen kommunizieren untereinander
  • Gabeln besitzen die Zustände "sauber" und "dreckig"
  • Druch eine einmalige initiale Zuweisung von Gabeln können Deadlocks und Starvation garantiert vermieden werden
